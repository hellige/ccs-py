Basic conjunction
---
frob = 'nope'
a b : frob = 'nitz'
---
a : frob = nope
b : frob = nope
a b : frob = nitz
b a : frob = nitz
===

Best before closest
---
first.id : test = "correct"
first second : test = "incorrect"
---
first.id second : test = correct
===

ID match has greater specificity
---
second.id : test = "correct"
first second : test = "incorrect"
---
first second.id : test = correct
===

Tied specificities
---
b.x : test = "foo"
c.x : test = "correct"
---
a b.x c.x : test = correct
===

Complex tie
---
b.x : test1 = "incorrect"
c.x : test1 = "correct"
c.x foo : test2 = "incorrect 1"
b.x foo : test2 = "correct"
foo : test2 = "incorrect 2"
---
b.x c.x : test1 = correct
b.x c.x foo : test2 = correct
===

Constraints at top level
---
@constrain a
test = "wrong"
a : test = "right"
---
test = right
===

Chained constraints at top level
---
@constrain a
test = "wrong"
a : @constrain b.c
b.c : test = "right"
---
test = right
===

Valueless constraint
---
a.b : @constrain c
c : test = "correct"
---
a.b : test = correct
===

Specific vs lowest
---
a.b c : test = "correct"
d : test = "incorrect"
---
a.b c d : test = correct
===

Specific vs lowest 2
---
a.b : test = "correct"
c : test = "incorrect"
---
a.b c d : test = correct
===

Multiple chained constraints and conjunction
---
test = "wrong"
a : @constrain b
a b : @constrain c
a b c : test = "right"
---
a : test = right
===

Bogus conflicts
---
a : @constrain b
a : @constrain c
a : test2 = "right"
---
a : test2 = right
===

Multiple chained constraints at root
---
@constrain a
a : @constrain b
a b : @constrain c
a b c : test = "right"
---
test = right
===

expect = true

env.lab, env.dev {
  expect = false

env.a env.dev : expect = true

env.prod : expect = true

Identical conj and disj
---
a b : foo = and
a, b : foo = or
---
a : foo = or
b : foo = or
a b : foo = and
===

Duplicated constraint builds incorrect dag
---
b a {}
a a : foo = here
---
a : foo = here
===

Override
---
a.b d : test = 'incorrect'
a : @override test = 'correct'
---
a.b d : test = correct
===

Override beats higher specificity
---
a b c : test = 'no'
a : @override test = 'yes'
---
a b c : test = yes
===

Override does not apply without match
---
a : @override test = 'overridden'
b : test = 'normal'
---
b : test = normal
===

Basic disjunction
---
a, b : test = 'either'
a b : test = 'both'
---
a : test = either
b : test = either
a b : test = both
===

Nested selectors
---
a {
  b {
    test = 'nested'
  }
  test = 'outer'
}
---
a : test = outer
a b : test = nested
===

Source order across rules
---
a : test = 'first'
a : test = 'second'
---
a : test = second
===

Disjunction re-activation
---
r, b.x : test = 'disj'
a : test = 'a_only'
---
a : test = a_only
a b.x : test = disj
===
